---
title: "TypeScript Path Alias in Next.js"
date: "2026-02-19"
tags: ["typescript", "nextjs"]
summary: "Use @/* to keep imports clean in app router projects."
draft: false
---

Set the alias in `tsconfig.json`:

```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

Then import modules like this:

```ts
import { getEntries } from "@/lib/content";
```

<Callout type="success" title="Result">
Imports become stable and easier to refactor.
</Callout>
